# Flight Data Analysis

## Project Snapshot
- Objective: Analyse large-scale flight data to identify factors affecting delays and performance  
- Techniques: Regression analysis (delay drivers) and clustering (route segmentation)  
- Tools: Python (Jupyter Notebook), R (RMarkdown)  
- Outcome: Identified key operational patterns and grouped flight routes with similar performance characteristics  

---

## Overview
This project explores large-scale flight datasets using both Python and R to uncover patterns in delays and operational performance. By combining statistical modelling and clustering techniques, the analysis identifies key drivers of delays and segments flight routes into meaningful groups for better decision-making.

---

## Dataset
- Dataset is large and not included in this repository  
- Source: [https://dataverse.harvard.edu/dataset.xhtml?persistentId=doi:10.7910/DVN/HG7NV7]

---

## Methodology
- Cleaned and prepared raw flight data for analysis  
- Performed exploratory data analysis (EDA) to detect trends and anomalies  
- Applied regression techniques to identify factors influencing flight delays  
- Used clustering methods to segment flight routes based on performance  
- Conducted analysis across both Python and R environments  

---

## 📈 Outputs
Analysis results are available within the Jupyter Notebooks and RMarkdown files, including visualisations and statistical outputs generated during the workflow.
---

## Key Insights
- Flight delays are influenced by multiple operational and route-specific factors  
- Distinct clusters of flight routes exhibit similar delay and performance patterns  
- Segmentation enables targeted operational improvements and better scheduling decisions  
- Insights support more efficient allocation of resources in flight operations
